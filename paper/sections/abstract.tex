Tool-using LLM agents require authorization at the tool boundary because prompts can induce filesystem and network side effects. We present Agent-Sentinel, a runtime mediation layer that interposes on every tool invocation and enforces capability and argument constraints before execution. We evaluate the system on a 200-scenario benchmark comprising 96 safe scenarios, 83 security-deny scenarios, and 21 robustness scenarios. Agent-Sentinel blocks 100.0\% of the security-deny scenarios while preserving 100.0\% of the safe scenarios. We compare against weaker defenses ranging from 50.6\% blocked for a static allowlist to 83.1\% for an argument allowlist, with Progent-style rules at 67.5\% and validator-only enforcement at 72.3\%. Median and p95 decision latency are 0.27\,ms and 0.67\,ms. These results position runtime capability mediation as a practical control layer for tool-using agents.
