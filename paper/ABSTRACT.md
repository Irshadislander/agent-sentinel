Tool-using AI agents improve task completion breadth but expand failure surfaces across policy bypass, ambiguous runtime errors, and incomplete forensic traces, especially when plugins and untrusted payloads are involved. This work presents a capability-enforced runtime model that treats safety and reliability as measurable system properties rather than qualitative goals. The method combines pre-execution policy gating, deterministic error taxonomy, structured trace emission, and controlled baseline ablations (`no_policy`, `no_trace`, `raw_errors`, `no_plugin_isolation`) over a fixed adversarial task suite. Evaluation is performed with a matrix benchmark and reproducible report generation using security-relevant metrics: Unsafe Execution Rate (UER), Failure Ambiguity Rate (FAR), Trace Completeness Ratio (TCR), and Exit Determinism Score (EDS), with supplemental plugin-load and latency statistics. In the current matrix results, disabling policy increased UER from 0.0 to 1.0 (+1.0), disabling trace reduced TCR from 1.0 to 0.0 (-1.0), and raw exception paths increased FAR from 0.0 to 1.0 (+1.0). These deltas validate that each enforcement component contributes distinct guarantees and provide a reproducible framework for comparing agent-runtime security controls under explicit threat assumptions.
