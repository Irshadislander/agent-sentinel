# Artifact Availability

To support reproducibility and artifact evaluation, we provide a complete implementation of Agent-Sentinel including all scripts, evaluation tasks, and experiment outputs required to reproduce the experiments reported in this paper.

The artifact package includes:

- Baseline evaluation suite used to measure policy enforcement across benign and adversarial tasks.
- Generated experiment outputs including evaluation tables and intermediate result files used in the paper.
- AutoGen integration case study demonstrating how Agent-Sentinel mediates tool calls generated by an autonomous agent workflow.
- Trace artifacts including JSONL logs produced by the flight-recorder component that records tool invocations, policy decisions, and blocked actions.
- Benchmark and task configurations including predefined benign and adversarial workloads.

Artifacts are organized in the repository as follows:

`artifacts/`
`examples/`
`paper/`

The artifacts directory contains experiment outputs and traces.  
The examples directory contains runnable evaluation harnesses and integration examples.  
The paper directory contains evaluation summaries and documentation.

An anonymized artifact distribution suitable for double-blind review can be prepared if required by the submission venue.

After publication, the artifact will be archived in a public repository and assigned a permanent identifier (e.g., DOI).
