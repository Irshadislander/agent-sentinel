# Figures

This document defines reviewer-facing figure intent and interpretation.

## Figure Descriptions

### 1) System Architecture Overview
- **Purpose**: show the main runtime components and trust boundaries in one view.
- **Variables plotted/shown**: component blocks (agent runtime, ToolGateway, policy store, capability map, tools, audit sink) and directed control/data flow edges.
- **Reviewer conclusion**: enforcement is performed at runtime boundaries with explicit policy and audit paths.

### 2) Capability Enforcement Pipeline
- **Purpose**: show deterministic request-to-decision flow before tool side effects.
- **Variables plotted/shown**: ordered stages: request intake -> capability lookup -> policy evaluation -> allow/deny decision -> artifact emission -> conditional tool execution.
- **Reviewer conclusion**: the runtime path is deterministic, fail-closed, and auditable.

### 3) Attack Family Blocking Summary
- **Purpose**: summarize security effectiveness across attack families and baselines.
- **Variables plotted**: x-axis attack family; y-axis ABR (optionally ASR); series/grouping by baseline.
- **Reviewer conclusion**: blocking behavior can be compared directly by family and baseline.

### 4) Security–Performance Tradeoff
- **Purpose**: visualize the relationship between security gain and runtime overhead.
- **Variables plotted**: x-axis latency overhead (typically p95/p99); y-axis security effectiveness (ABR or ASR delta); optional color/marker for baseline or observability quality.
- **Reviewer conclusion**: security improvements are achieved with bounded overhead, and weak baselines may trade safety for speed.

### 5) Scaling Curve
- **Purpose**: show performance behavior as workload size grows.
- **Variables plotted**: x-axis workload size; y-axis throughput and/or p95 latency; one line per system condition.
- **Reviewer conclusion**: scaling behavior is observable and comparable under increasing load.
